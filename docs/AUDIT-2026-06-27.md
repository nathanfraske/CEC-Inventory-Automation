## Fixed this pass (commit on branch `claude/runbook-setup-config-oxwath`)

- ✅ **[CRITICAL] Object store not backed up.** `db_backup.sh` dumped only Postgres; receipts
  (the legal RMA proof artifacts, scope §7) live in a separate volume nothing touched — a
  restore left dangling `receipt_files` pointers. Now `db_backup.sh` archives the object store
  timestamp-paired with the DB dump, validates the dump (`pg_restore --list`), uses
  `set -euo pipefail`; `db_restore.sh` restores the paired objects archive and documents the
  restore-vs-migrations order.
- ✅ **[HIGH] Forgeable sessions on misconfig.** `SESSION_SECRET` had a committed dev-default
  fallback, silently zero-padded to 64 bytes (a known signing key) and weakening short secrets.
  Now fails closed (no default) and requires ≥64 bytes — no zero-pad. CI `tests` job supplies a
  throwaway value. (`crates/api/src/lib.rs`)
- ✅ **[HIGH] No request body-size limit.** Added a global 1 MiB cap (DoS guard) + a 25 MiB
  per-route cap on the receipt/image upload routes (also fixes the latent "2 MB default rejects
  real photos" bug). (`crates/api/src/lib.rs`, `routes/mod.rs`)
- ✅ **[HIGH] Reserve/consume race.** `direct.rs::transition` took `FOR UPDATE` on the pool, so
  the lock released before the write — two concurrent calls could double-reserve. Lock now lives
  inside the transaction. (`crates/api/src/routes/direct.rs`)
- ✅ **[HIGH] Event log written outside a tx.** `rma.rs::update_rma` and `scan.rs::unit_label`
  did mutation + event as two autocommits (event could be lost). Both now wrap mutation + event
  in one transaction with a row lock; `unit_label` only logs on a genuinely new tag.
- ✅ **[HIGH] Operator-regex resource use.** `serial_format_regex` is now compiled with a size
  limit (`RegexBuilder.size_limit/dfa_size_limit`). (`crates/api/src/routes/scan.rs`)
- ✅ **[HIGH/MED] Unvalidated extraction line items.** `persist_extraction` now caps line-item
  count + raw payload size and rejects negative/absurd quantities and negative money — protects
  the `from-payload` and VLM paths. (`crates/api/src/extractor.rs`)
- ✅ **[LOW] Input hardening.** `from-image` whitelists the media type and caps `vendor_hint`/
  `created_by`; password floor raised 8→12; login does a dummy argon2 verify on unknown users
  (no timing enumeration). (`extractor.rs`, `auth.rs`)

## Security panel — open backlog (prioritized)

- ⬜ **[HIGH] No login/bootstrap rate-limiting or lockout.** Brute-force is unthrottled; the
  `bootstrap` endpoint re-arms whenever `app_user` is empty. → add `tower_governor` on `/auth/*`
  + a failed-attempt counter; consider a bootstrap token instead of an open endpoint.
- ⬜ **[MED] Sessions never expire / can't be revoked.** Cookie carries the bare user id, no TTL,
  logout is client-side only. → embed issued-at/expiry (or a `session` table for server-side
  revocation); rotate on login/password-change.
- ⬜ **[MED] Flat authorization + spoofable actor.** Every authenticated user is fully privileged
  (incl. user-creation and full `/export`); the `unit_event.actor` is taken from the request
  body. → add a `role` column + gate admin/export; derive `actor` from the session.
- ⬜ **[MED] Prompt-injection → poisoned extraction → latent stored XSS.** Persisted extractor
  text isn't rendered today (no purchase/line-item detail view), so it's latent — but any future
  detail page MUST route persisted strings through `esc()`; the `raw_extract` jsonb must never be
  dumped raw into HTML.
- ⬜ **[LOW] CSRF rests solely on `SameSite=Lax`** (holds today, esp. for JSON routes). → add a
  CSRF token for the multipart routes as defense-in-depth.
- ⬜ **[LOW] Argon2 uses library-default params.** → pin explicit `Params` sized to the box.
- ⬜ **[LOW] Containers run as root; no resource limits / `cap_drop` / read-only FS.** → add a
  non-root `USER` to all three Dockerfiles + compose `mem_limit`/`pids_limit`/`cap_drop:[ALL]`/
  `no-new-privileges`/`read_only` with explicit writable mounts.
- ⬜ **[LOW] No dependency-advisory scanning; base images pinned by mutable tag.** → add CI
  `cargo audit`/`cargo-deny` + `pip-audit` + `.github/dependabot.yml`; pin base images (and GH
  Actions) by digest; pin the Python web deps exactly. (`rsa 0.9.10` advisory is present via
  sqlx-mysql but off the postgres path — deny-list/ignore it explicitly.)
- ⬜ **[LOW] Vision egress has no audit log / size cap / PII note.** → log each outbound vision
  call, cap image size before egress, document the PII-leaves-boundary implication in
  `SECRETS-AND-DATABASE.md`.
- ⬜ **[INFO] DB port published to host loopback** though services use the compose network — drop
  the `db` `ports:` block in prod.

### Security — verified done well
Parameterized SQL throughout (the `format!`-built queries interpolate only `'static` column
consts; all user data is `.bind()`-bound — no SQLi); consistent `esc()` HTML escaping (no
reflected/DOM XSS); no SSRF (egress hosts are env-fixed); strong secret hygiene
(gitignore/dockerignore/gitleaks/pre-commit/CI, no secrets in image layers, key-from-env);
argon2id password hashing; signed `HttpOnly` cookies; sane healthchecks; path-traversal-safe
uploads (`sanitize_filename` + UUID prefix); rustls TLS; dependencies on current advisory-clean
majors.

## Data-integrity / backups panel — open backlog (prioritized)

- ✅ **[HIGH] Serial-number uniqueness.** FIXED (migration 0003, D-017): owner chose **globally
  unique** — partial unique index on `serial_number WHERE NOT NULL`; duplicate bind/intake → 409.
- ✅ **[MED] Append-only log DB enforcement.** FIXED (migration 0003): `BEFORE UPDATE OR DELETE`
  trigger on `unit_event`, `system_validation`, `system_transfer`, `shipment_event` that raises.
- ✅ **[MED] System-gating TOCTOU.** FIXED: a `lock_system` helper takes `SELECT … FOR UPDATE`
  on the `system` row as the first statement inside the tx for `add_member`/`remove_member`/
  `validate`/`deliver`/`sweep`/`transfer`; gates re-read the locked state, and member/snapshot
  reads moved inside the tx. Concurrent ops serialize on the row.
- ✅ **[MED] Unguarded status transitions.** FIXED: `units.change_status` enforces an
  allowed-transition matrix (`scrapped` terminal, `returned` must restock first, …);
  `rma.update_rma` rejects moving a terminal `closed` case to another status.
- ✅ **[MED] `/export` completeness.** FIXED: added `vendor_return_policy`, `cec_warranty_policy`,
  `trade_in`, `trade_in_unit`. `app_user` is **deliberately excluded** (credentials); documented
  that `pg_dump` + the object-store archive is the authoritative backup (receipt bytes aren't
  inlined).
- ◐ **[MED] `asset_tag` uniqueness.** PARTIAL (migration 0003): `stock_item.asset_tag` is now
  `UNIQUE` (units/systems already were). Cross-table global tag uniqueness still open.
- ⬜ **[HIGH] Backups are manual, unencrypted, unrotated, with no tested restore.** (Scripts now
  cover both data stores — see Fixed.) → schedule (systemd timer/cron), retention/rotation,
  encrypt at rest (age/gpg or restic/borg), real offsite over the mesh, and a CI/timer restore
  drill. Enable WAL archiving/PITR if RPO must beat the dump cadence.
- ⬜ **[LOW] f64→Decimal money in the extractor path is lossy** (vs string-parsed money in
  `purchases.rs`). → parse extractor money from strings.
- ⬜ **[LOW] Inconsistent FK delete policy; `unit_event` cascades on unit delete** (would erase
  history — no delete handler exists today). → set audit-table FKs to `RESTRICT` and document the
  never-delete/tombstone policy.
- ⬜ **[INFO] Policy lookups (`vendor_return_policy`/`cec_warranty_policy`) are nondeterministic**
  without `UNIQUE (warranty_class, category)`. → add the unique constraint.

### Data-integrity — verified done well
Event-in-same-tx holds for the core mutators (units, intake, direct, systems deliver/transfer/
membership, rma open/replacement, costing, warranty); no code mutates `unit_event`;
`stock.adjust` and `units.change_status` lock-and-write correctly; child/audit FKs cascade from
parents; money in purchases/costing uses `Decimal` with a remainder-on-last-line allocator that
sums to the whole; `app_user.username` is unique; uploads are traversal-safe.
